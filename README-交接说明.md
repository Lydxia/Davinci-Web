>    - **回收公式**:shader 里 `mod(...)-uLen` 给的是**绝对** z∈[-260,0],没跟着相机走。改成相机相对:`p.z = -uCamZ - phase`(= camZ - phase),粒子恒在相机前方一个 `uLen` 走廊里循环。

## 这是什么
给「达芬奇机器人」(Da Vinci Robotics,自主巡检机器狗公司)做的沉浸式 3D 滚动网站。
单页面,纯前端,无框架。核心是 Three.js 做的"机器狗钻进隧道、隧道穿行中依次展示应用场景"的滚动叙事。

## 怎么构建(重要)
**不要直接编辑 `davinci-v3-preview.html`** —— 它是构建产物(把所有东西内联打包成的成品,3.4MB)。

**要改就改 `index.html`(源文件),然后运行构建:**
```
cd 项目目录
python3 build.py
```
`build.py` 会:
1. 把 `three.global.js` 内联进去(Three.js 的全局版本,window.THREE)
2. 把源文件里的 ES module 脚本转成普通 IIFE
3. 把字体 `font-900.b64` / `font-700.b64` 内联(替换 `__FONT900__`/`__FONT700__` 占位符)
4. 把 `assets/` 里的图片转 base64 内联(替换 `src="assets/X"` 和 `'assets/X'`)
5. 输出 `davinci-v3-preview.html`(完全自包含,可直接 file:// 打开,也可拖到 Netlify Drop 部署)

部署:把产物复制成 `index.html`,拖到 app.netlify.com/drop 即可生成链接。

## 技术要点
- 整个 3D 是**单个 Three.js 场景**,相机随滚动沿 -Z 前进(`camZ`)。所有阶段由滚动进度 `p`(0-1,基于 **1050vh** 的 spacer 高度)驱动。
- 时间轴在 `tick()` 函数里:`HERO_END=0.11`(首屏+钻隧道,已缩短),之后 `ps` 驱动内容段。`ps` 时间轴排布:
  - `ps 0.02–0.31`:四拍 intro 序列(每拍 `BEATSPAN=0.072`,共 4 拍;`active=floor((ps-BEAT0)/BEATSPAN)`)
  - `ps 0.32–0.355`:intro 退场
  - `ps 0.375–0.49`:大标题进出
  - `ps 0.50–0.995`:六个场景(`START=0.50`)
  - 改时间轴时注意这几段**首尾不要重叠**(已用数值模拟校验过,见对话记录)。
- 场景框是 **HTML overlay**(不是 3D 物体),用 CSS 3D `translateZ` + perspective 做纵深飞掠。intro 四拍也是 overlay(`.ti-beat` 绝对定位叠放,靠 opacity 交叉淡入)。

## 字体重新子集化方法(新增中文文案时必看)
页面中文用思源黑体(Noto Sans SC)的**子集化**字体,只含当前页面用到的字(目前 264 个)。
**如果新增了页面里还没出现过的中文字,不重新子集化就会掉成系统字体、字形不一致。**

为方便交接,本包 `fonts-full/` 里已附**原始完整字重**(Heavy 900 / Bold 700,各约 1.1MB,全字库),
所以重新子集化是**自包含的,不用再去 npm 下载**。完整流程(在项目根目录):

```bash
# 0) 一次性装依赖
pip install fonttools brotli

# 1) 收集页面当前所有中文字 -> chars.txt
python3 - <<'PY'
import re
html=open('index.html',encoding='utf-8').read()
chars=sorted(set(re.findall(r'[\u4e00-\u9fff]',html)))
open('chars.txt','w',encoding='utf-8').write(''.join(chars))
print('chars:',len(chars))
PY

# 2) 子集化两个字重(Heavy 900 / Bold 700)
pyftsubset fonts-full/noto-sans-sc-chinese-simplified-900-normal.woff2 \
  --text-file=chars.txt --flavor=woff2 --output-file=sub-900.woff2 \
  --layout-features='' --no-hinting --desubroutinize
pyftsubset fonts-full/noto-sans-sc-chinese-simplified-700-normal.woff2 \
  --text-file=chars.txt --flavor=woff2 --output-file=sub-700.woff2 \
  --layout-features='' --no-hinting --desubroutinize

# 3) base64 编码,覆盖 font-900.b64 / font-700.b64
base64 -w0 sub-900.woff2 > font-900.b64
base64 -w0 sub-700.woff2 > font-700.b64

# 4) 重新构建
python3 build.py
```

> 完整字体的来源是 npm 包 `@fontsource/noto-sans-sc`(`package/files/` 下的
> `noto-sans-sc-chinese-simplified-{700,900}-normal.woff2`)。万一 `fonts-full/` 丢了,
> `npm pack @fontsource/noto-sans-sc` 解开就能再拿到。
>
> 校验小技巧:子集后可以用 fontTools 读 cmap 数量确认覆盖,例如
> `python3 -c "from fontTools.ttLib import TTFont; print(len(TTFont('sub-900.woff2').getBestCmap()))"`
> 数字应等于 chars.txt 的字数。
